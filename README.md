## What I build

**Tooling** — CLI/TUI apps, automation workflows, file-sync systems and developer utilities.  
**Web products** — landing pages, dashboards, education apps and applied interfaces.  
**Minecraft tooling** — modpack infrastructure, updater systems, plugins and server utilities.  
**Education tools** — focused products for real study workflows, exam prep and repetition.

I like projects where software is not just a demo, but a controlled workflow: clear UX, reproducible setup, safe defaults and honest documentation.

## Selected work

### 🛡️ PackGuard

Safety-first manifest sync toolkit for Minecraft modpack maintainers.

Focuses on manifest validation, dry-run planning, guarded apply, integrity checks and release discipline.

`TypeScript` `CLI` `Safety model` `Minecraft tooling`

**Status:** v0.1 alpha  
**Repository:** [KROFN/packguard](https://github.com/KROFN/packguard)

`Python` `TUI` `CLI` `Automation`
